# Roadmap

Postbox начинался как Telegram-бот. Он помог проверить предметную модель и собрать полный цикл письма:
отправить, получить, найти в журнале, отметить доставку и сохранить заметку.

Бот остаётся рабочим прототипом. Следующая форма Postbox — небольшая mobile-first PWA для iPhone и
Android. Она открывается по ссылке и при желании устанавливается на домашний экран без App Store и
Google Play.

## Основное решение

- Делаем один web-интерфейс для iPhone и Android.
- Размещаем PWA, Python и PostgreSQL на существующем VPS.
- Сохраняем текущую предметную модель и Active Record.
- Оставляем Telegram-бота резервным интерфейсом на время перехода.
- Входим в Postbox по личному приглашению.
- Не готовимся к магазинам приложений.
- Не храним фотографии.

```text
iPhone / Android → Postbox PWA → Python API → PostgreSQL
```

## Что уже есть

- [x] приватные пользователи;
- [x] входящие и исходящие письма;
- [x] неизвестная дата отправки входящего письма;
- [x] отметка доставки;
- [x] журнал и карточка письма;
- [x] заметки;
- [x] PostgreSQL и миграции.

Эту основу не переписываем. Меняется интерфейс, а не смысл продукта.

## Шаг 1. Собрать мобильную оболочку

Сначала делаем интерфейс с тестовыми данными, не трогая сервер.

Основные экраны:

1. Дом — письма в пути и последние события.
2. Журнал — вся почта по времени.
3. Карточка письма — человек, даты, статус, путь и заметка.
4. Новое письмо — отправленное или неожиданно полученное.

- [ ] спроектировать всё сначала для ширины телефона;
- [ ] собрать навигацию и состояния экранов;
- [ ] проверить пустой журнал, длинные имена и неизвестные даты;
- [ ] найти типографику, ритм и атмосферу Postbox;
- [ ] открыть прототип на iPhone и Android.

**Проверка:** хочется ли возвращаться в этот интерфейс как в коробку с письмами? Если нет, исправляем форму
и не строим следующую часть.

## Шаг 2. Подключить настоящие данные

- [ ] открыть существующие операции через небольшой HTTP API;
- [ ] сохранить проверку владельца для каждой записи;
- [ ] добавить вход по одноразовому приглашению;
- [ ] подключить журнал и карточки;
- [ ] подключить отправку, получение, доставку и заметки;
- [ ] добавить manifest, иконки и standalone-режим PWA;
- [ ] показать понятное состояние при плохой сети;
- [ ] покрыть API интеграционными тестами.

Полноценная публичная регистрация, email, восстановление пароля, offline-first и push-уведомления пока не
нужны.

**Проверка:** удобнее ли PWA текущего бота на одном iPhone?

## Шаг 3. Установить семейную альфу

- [ ] развернуть PWA и API на VPS за HTTPS;
- [ ] настроить миграции, ежедневный backup PostgreSQL и журнал ошибок;
- [ ] добавить Postbox на домашний экран iPhone;
- [ ] отправить приглашение на Android и добавить Postbox на его домашний экран;
- [ ] проверить обновление интерфейса без повторной установки;
- [ ] несколько недель записывать только реальные затруднения.

**Проверка:** может ли второй человек самостоятельно открыть приглашение и записать письмо? Если да,
выбранного способа распространения достаточно.

## После семейной альфы

Добавляем только то, что стало нужно в жизни:

- история писем по человеку;
- исправление и удаление ошибочных записей;
- поиск и фильтры;
- экспорт личного архива;
- страны и статистика доставки;
- спокойные web push-уведомления;
- фотографии, если появится понятный способ хранения и резервного копирования.

Нативное приложение и магазины рассматриваются только тогда, когда обнаружится конкретное ограничение
PWA. Заранее планировать их не нужно.

## Последовательность изменений

1. `feat: prototype mobile postbox shell`
2. `refactor: isolate postbox domain from telegram`
3. `feat: expose private postbox api`
4. `feat: connect postbox pwa`
5. `chore: deploy private family alpha`

Путь короткий: сначала форма, затем настоящие данные, затем два телефона. Всё остальное может подождать.
